How to fork the GitHub repository

  1. Create a GitHub account
  2. Create your own "fork" of the readordie repository by going to the repository and clicking the Fork button.


    3. Location of the Fork button on a different repo. It'll be in the same spot.

  3. Go to your new fork of the repo and clone that repository onto your computer with the Clone or download button.

  4. Use Terminal to cd into your newly cloned repo and enter git status, which should tell you you are up-to-date! Now you are ready to add a post.

How to make a post

  1. Make sure you've forked the readordie repository.
  2. Posts are written using Markdown syntax. The only constraint beyond that is that you must place the file in the _posts directory, follow the filename template, and insert a proper header in the file.

    3. Filenames must follow this format:
    <year>-<month>-<day>-<lead_author>-<pub_year>-<journal_abbrev>.md

    Headers must be inserted at the top of the post file and follow this format:
     ---
     layout: post
     title: "Fogel et al., 2018, Front Psychol"
     author: remrama
     date: 2019-08-19 16:04:08 -0500
     tags: [dreaming,textanalysis]
     article_title: "A Novel Approach to Dream Content Analysis Reveals Links Between Learning-Related Dream Incorporation and Cognitive Abilities"
     article_doi: https://doi.org/10.3389/fpsyg.2018.01398
     ---

How to test your post locally.

  1. Install this:  gem install github-pages
  2. Enter this from your repository directory: jekyll build; jekyll serve --baseurl ''
  3. Open your browser and go to http://localhost:4000
  4. Refresh this page as you update your post.
